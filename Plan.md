# Plan · SceneWeaver（小说转剧本助手）

> 朝前看的计划与进度。领域方法论（怎么改编、剧本格式知识）不写这里，已归档到个人知识库 `OS second brain/wiki/剧本改编/`。

## 一句话

把 3 章以上小说自动改编成**结构化、可溯源、可打磨**的剧本 YAML；定位"给作者的结构化改编助手"，区别于黑盒一键量产工具。

## 评分对齐（赛规权重）

| 维度 | 权重 | 我们怎么拿分 |
|---|---|---|
| 完整度 + 创新性 | 40% | 端到端可用 + 结构化/溯源/置信度/双阶段 四个差异化点 |
| 工程过程与质量 | 40% | 双阶段清晰架构 + Schema 设计文档 + 分 PR、commit 分布到 3 天 |
| 演示与表达 | 20% | 左原文右剧本的编辑器，点场景跳回原文，录 demo |

## 架构（双阶段 pipeline）

1. **理解层**：长文分块 → 抽人物表(别名归一) / 场景切分 / 情节 beat → 中间表示(IR)。
2. **改编层**：IR → 视听化（删内心戏、外化动作、重写对白）→ 剧本 YAML（见 `docs/剧本Schema设计文档.md`）。
3. **编辑层**：左原文右剧本，源文映射，低置信标红。
